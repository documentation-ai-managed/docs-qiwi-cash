<Update label="2024-09-20" description="v2.0.0" tags={["feature", "breaking"]}>

## New Features

- Launched Qiwi Cash Dashboard with analytics, transaction history, and export tools.
- Added OAuth 2.0 authentication flow for secure partner integrations.

## Breaking Changes

- Updated API base URL from `/v1` to `/v2`. Update your endpoints accordingly.
- Deprecated `amount_cents` parameter; use `amount` with `currency` instead.

<CodeGroup tabs="JavaScript,Python">
```javascript
// Old v1
fetch('https://api.example.com/v1/transactions', {
  body: JSON.stringify({amount_cents: 1000})
});

// New v2
fetch('https://api.example.com/v2/transactions', {
  body: JSON.stringify({amount: 10, currency: 'USD'})
});
```
```python
# Old v1
requests.post('https://api.example.com/v1/transactions',
              json={'amount_cents': 1000})

# New v2
requests.post('https://api.example.com/v2/transactions',
              json={'amount': 10, 'currency': 'USD'})
```
</CodeGroup>

## Bug Fixes

- Corrected token refresh logic in SDKs.

</Update>

## How to Upgrade

Follow these steps to upgrade Qiwi Cash safely.

<Steps>
  <Step title="Review Changes" icon="book-open">
    Check the release notes above for breaking changes.
  </Step>
  <Step title="Backup Data" icon="database">
    Export your transaction history from the dashboard.
  </Step>
  <Step title="Update SDK" icon="download">
    Run `npm update qiwi-cash-sdk` or equivalent for your language.
  </Step>
  <Step title="Test Endpoints" icon="test-tube">
    Verify API calls to `https://api.example.com/v2/transactions`.
  </Step>
</Steps>

## Deprecated Features

- `GET /v1/balances` endpoint – Migrate to `GET /v2/wallets`.
- Legacy callback URLs – Switch to webhooks for better reliability.
